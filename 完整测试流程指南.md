## 验证 DRL 路径是否生效

### 方法 1：查看 Controller 日志

**在 Controller 终端（终端 2）中查找**：
```
【DRL路径】1 -> 2: [1, 3, 4, 2]
```

**如果看到 "【DRL路径】"，说明使用了 DRL 模型。**

**如果看到 "【最短路径】"，说明回退到了最短路径算法。**

---

### 方法 2：查看 Path_service 日志

**在 Path_service 终端（终端 1）中查找**：
```
→ 收到路径计算请求: 0 -> 1
✓ 返回路径: [0, 2, 3, 1]
```

**如果看到这些日志，说明路径计算服务正常工作。**

---

### 方法 3：对比路径差异

**DRL 路径**（考虑网络状态）：
```
节点 0 -> 节点 1: [0, 2, 3, 1]  (可能绕路，但性能更好)
```

**最短路径**（仅考虑跳数）：
```
节点 0 -> 节点 1: [0, 1]  (直接连接)
```

**如果 DRL 路径与最短路径不同，说明 DRL 模型在工作！**

## 故障排查

### 问题 1：Controller 日志显示 "【最短路径】"

**可能原因**：
1. Path_service 未启动
2. Path_service 连接失败
3. DRL 功能被禁用

**解决方法**：
1. 检查 Path_service 是否运行（终端 1）
2. 检查端口 8889 是否监听：`netstat -tulpn | grep 8889`
3. 检查 Controller 日志中的错误信息

---

### 问题 2：Path_service 未收到请求

**可能原因**：
1. Controller 的 `get_path` 方法未被调用
2. DRL 功能被禁用（`self.drl_enabled = False`）

**解决方法**：
1. 检查 Controller 日志，确认有流量到达
2. 检查 `get_path` 方法是否被调用
3. 确认 `self.drl_enabled = True`

---

### 问题 3：路径计算失败

**可能原因**：
1. 模型文件不存在或损坏
2. 拓扑不匹配
3. 网络状态异常

**解决方法**：
1. 检查模型文件：`ls -la drl-or-s/model/.../agent0.pth`
2. 检查 Path_service 日志中的错误信息
3. 查看完整的 traceback

---

### 问题 4：Mininet 无法连接控制器

**错误信息**：
```
Unable to contact the remote controller at 127.0.0.1:5001
```

**解决方法**：
1. 确认 Controller 已启动
2. 确认 Controller 使用 `--ofp-tcp-listen-port 5001`
3. 检查端口是否被占用：`netstat -tulpn | grep 5001`
