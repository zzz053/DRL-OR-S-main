# 控制器接口需求文档

## 📋 概述

本文档定义了 DRL-OR-S 项目中控制器必须满足的接口要求。**关键点：testbed.py 不使用 REST API，而是通过 OpenFlow 协议和 Socket 通信与控制器交互。**

## 🔌 接口 1：OpenFlow 控制平面（Mininet ↔ 控制器）

### 监听地址

- **协议**: OpenFlow 1.3 (协议版本号 0x04)

### 必须处理的 OpenFlow 事件

#### 1. **EventOFPSwitchFeatures**（交换机连接事件）
- **触发时机**: 当 OVS 交换机连接到控制器时
- **必须操作**:
  - 保存 datapath 对象（用于后续流表安装）
  - 安装 table-miss 流表项（优先级 0，未匹配时发送到控制器）
- **Ryu 示例**（controller.py 第 136-154 行）:
```python
@set_ev_cls(ofp_event.EventOFPSwitchFeatures, CONFIG_DISPATCHER)
def switch_features_handler(self, ev):
    datapath = ev.msg.datapath
    dpid = datapath.id
    self.datapaths[dpid] = datapath  # 必须保存！
    
    # 安装 table-miss 流表项
    match = parser.OFPMatch()
    actions = [parser.OFPActionOutput(ofproto.OFPP_CONTROLLER, ofproto.OFPCML_NO_BUFFER)]
    self.add_flow(datapath, 0, match, actions)
```

#### 2. **EventOFPPacketIn**（数据包上送事件）
- **触发时机**: 交换机收到未匹配流表的数据包
- **必须处理的包类型**:
  - **ARP 请求/应答**（必须处理，否则主机无法通信）
    - **处理方式**: 向所有交换机的主机端口（host port）泛洪
  - **LLDP 数据包**（应忽略）
  - **其他未匹配数据包**（可记录日志或丢弃）

- **Ryu 示例**（controller.py 第 176-216 行）:
```python
@set_ev_cls(ofp_event.EventOFPPacketIn, MAIN_DISPATCHER)
def _packet_in_handler(self, ev):
    # 解析数据包
    pkt = packet.Packet(msg.data)
    eth = pkt.get_protocols(ethernet.ethernet)[0]
    
    # 忽略 LLDP
    if eth.ethertype == ether_types.ETH_TYPE_LLDP:
        return
    
    # 处理 ARP（关键！）
    arppkt = pkt.get_protocol(arp.arp)
    if arppkt != None:
        # 向所有主机端口泛洪
        for nodeid in range(self.node_num):
            actions = [parser.OFPActionOutput(self.switch_host_port[nodeid + 1])]
            datapath = self.datapaths[nodeid + 1]
            out = parser.OFPPacketOut(
                datapath=datapath,
                buffer_id=ofproto.OFP_NO_BUFFER,
                in_port=ofproto.OFPP_CONTROLLER,
                actions=actions, data=msg.data)
            datapath.send_msg(out)
```

## 🔌 接口 2：路径下发接口（DRL Agent ↔ 控制器）

### 监听地址
- **IP**: `127.0.0.1`
- **端口**: `3999`
- **协议**: TCP Socket
- **数据格式**: JSON

### 代码位置
**controller.py 第 79-131 行：**
```python
def install_rules(self):
    TCP_IP = "127.0.0.1" 
    TCP_PORT = 3999
    BUFFER_SIZE = 1024
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.bind((TCP_IP, TCP_PORT))
    s.listen(1)
    
    self.simenv_socket, addr = s.accept()
    print("Connection address:", addr)
    
    # 等待所有交换机连接
    while len(self.datapaths) < self.node_num:
        hub.sleep(5)
    
    # 接收路径并安装流表
    while True:
        msg = self.simenv_socket.recv(BUFFER_SIZE)
        data_js = json.loads(msg.decode('utf-8'))
        # ... 安装流表 ...
        self.simenv_socket.send("Succeeded!".encode())
```

### 请求数据格式

**从 DRL Agent 接收的 JSON 数据结构：**
```json
{
    "path": [0, 2, 3, 5],
    "src_port": 10001,
    "dst_port": 10002,
    "ipv4_src": "10.0.0.1",
    "ipv4_dst": "10.0.0.6"
}
```

**字段说明：**
| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `path` | `list[int]` | 路由路径（节点 ID 列表，从 0 开始索引） | `[0, 2, 3, 5]` 表示 节点1→节点3→节点4→节点6 |
| `src_port` | `int` | UDP 源端口 | `10001` |
| `dst_port` | `int` | UDP 目的端口 | `10002` |
| `ipv4_src` | `string` | 源 IP 地址 | `"10.0.0.1"` |
| `ipv4_dst` | `string` | 目的 IP 地址 | `"10.0.0.6"` |

**注意事项：**
- `path` 中的节点 ID **从 0 开始索引**，但 OpenFlow datapath ID **从 1 开始**
  - 例如：`path[0] = 0` 对应 `dpid = 1`
- 协议类型固定为 **UDP (IP proto = 17)**

### 响应数据格式

**控制器必须返回的确认消息：**
```
"Succeeded!"
```
- **类型**: UTF-8 编码的字符串
- **长度**: 10 字节

### 必须安装的流表规则

#### 五元组匹配规则
对路径中的每个节点，安装如下流表项：

**匹配字段（OpenFlow Match）：**
```python
match = parser.OFPMatch(
    ipv4_src=ipv4_src,      # 源 IP
    udp_src=src_port,        # UDP 源端口
    ipv4_dst=ipv4_dst,       # 目的 IP
    udp_dst=dst_port,        # UDP 目的端口
    ip_proto=17,             # UDP 协议
    eth_type=0x0800          # IPv4
)
```

**动作（Actions）：**
- **中间节点**: 转发到下一跳节点的端口
- **目的节点**: 转发到主机连接端口（host port）
- **环路检测**: 如果路径中出现环路，则安装 DROP 动作

**流表优先级：**
- **动态业务流表**: 优先级 **5**
- **Table-miss 规则**: 优先级 **0**

#### 端口映射逻辑
控制器需要维护两个映射表：

**1. `link_port[dpid][next_dpid]`** - 交换机之间的互联端口
```python
# 示例：拓扑为 s1-s2-s3-s4（环形）
link_port = {
    1: {2: 1, 4: 2},  # s1 到 s2 用端口 1，到 s4 用端口 2
    2: {1: 1, 3: 2},
    3: {2: 1, 4: 2},
    4: {3: 1, 1: 2}
}
```

**2. `switch_host_port[dpid]`** - 交换机到主机的端口
```python
# 示例
switch_host_port = {
    1: 3,  # s1 的主机在端口 3
    2: 3,
    3: 3,
    4: 3
}
```

**构建逻辑**（controller.py 第 58-70 行）：
```python
self.link_port = {}
for i in range(self.node_num):
    self.link_port[i + 1] = {}

switch_port_counter = [1] * self.node_num
for link in self.linkset:
    u, v = link
    self.link_port[u + 1][v + 1] = switch_port_counter[u]
    self.link_port[v + 1][u + 1] = switch_port_counter[v]
    switch_port_counter[u] += 1
    switch_port_counter[v] += 1

self.switch_host_port = {}
for i in range(self.node_num):
    self.switch_host_port[i + 1] = switch_port_counter[i]
```

#### 流表安装代码示例
**controller.py 第 106-128 行：**
```python
temp = {}
for i in range(len(path)):
    temp[path[i]] = 1
    
    dpid = path[i] + 1  # 节点 ID 转 dpid
    datapath = self.datapaths[dpid]
    ofproto = datapath.ofproto
    parser = datapath.ofproto_parser
    match = parser.OFPMatch(
        ipv4_src=ipv4_src, udp_src=src_port, 
        ipv4_dst=ipv4_dst, udp_dst=dst_port, 
        ip_proto=17, eth_type=0x0800
    )
    
    # 环路检测
    if i < len(path) - 1 and path[i + 1] in temp:
        actions = []  # DROP
        self.add_flow(datapath, self.action_rule_priority, match, actions)
        break
    
    # 计算输出端口
    if i == len(path) - 1:
        out_port = self.switch_host_port[dpid]  # 最后一跳到主机
    else:
        out_port = self.link_port[dpid][path[i + 1] + 1]  # 到下一跳
    
    actions = [parser.OFPActionOutput(out_port)]
    self.add_flow(datapath, self.action_rule_priority, match, actions)
```

## 🗂️ 拓扑信息加载

控制器需要从 `../topology/{toponame}/Topology.txt` 加载拓扑信息。

### 拓扑文件格式
**第一行**: `nodeNum linkNum`
**后续行**: `u v weight capacity loss`

**示例**（topology/Abi/Topology.txt）：
```
11 14
1 4 1176 9920 0
1 5 587 2480 10
2 3 1034 9920 0
...
```

### 加载代码
**controller.py 第 29-37 行：**
```python
def load_topoinfo(toponame):
    topo_file = open("../topology/%s/Topology.txt" % (toponame))
    content = topo_file.readlines()
    nodeNum, linkNum = list(map(int, content[0].split()))
    linkSet = []
    for i in range(linkNum):
        u, v, w, c, loss = list(map(int, content[i + 1].split()))
        linkSet.append([u - 1, v - 1])  # 转换为 0-based 索引
    return nodeNum, linkSet
```

## ✅ 控制器兼容性检查清单

在替换控制器前，请确保您的实现满足以下要求：

### 基础要求
- [ ] 支持 OpenFlow 1.3 协议
- [ ] 监听在 `127.0.0.1:5001`（OpenFlow）
- [ ] 监听在 `127.0.0.1:3999`（Socket）
- [ ] 能够从 `../topology/{name}/Topology.txt` 加载拓扑

### OpenFlow 功能
- [ ] 处理 `EventOFPSwitchFeatures`（保存 datapath 对象）
- [ ] 安装 table-miss 流表项（优先级 0）
- [ ] 处理 `EventOFPPacketIn`
- [ ] **必须处理 ARP 泛洪**（否则主机无法通信）
- [ ] 忽略 LLDP 数据包

### 路径下发功能
- [ ] 接受包含 `path`, `src_port`, `dst_port`, `ipv4_src`, `ipv4_dst` 的 JSON
- [ ] 正确转换节点 ID（0-based）到 dpid（1-based）
- [ ] 安装五元组匹配规则（src_ip, src_port, dst_ip, dst_port, protocol）
- [ ] 正确计算输出端口（使用 `link_port` 和 `switch_host_port`）
- [ ] 检测并处理路径环路（安装 DROP 规则）
- [ ] 返回 `"Succeeded!"` 确认消息

### 性能要求
- [ ] 等待所有交换机连接后再接受路径下发请求
- [ ] 流表安装延迟 < 100ms（建议）

## 🚨 常见错误

### 错误 1：主机无法 ping 通
**原因**: 控制器未正确处理 ARP 请求
**解决**: 确保 ARP 数据包被泛洪到所有主机端口

### 错误 2：数据包丢失
**原因**: 
- 流表未安装成功
- 输出端口计算错误（混淆了 dpid 和 node_id 索引）
**解决**: 
- 检查 `link_port` 和 `switch_host_port` 映射
- 确保 `path` 中的节点 ID 加 1 才是 dpid

### 错误 3：控制器连接失败
**原因**: 监听端口错误或 IP 地址配置错误
**解决**: 
- OpenFlow 端口必须是 `5001`
- Socket 端口必须是 `3999`
- 必须监听 `127.0.0.1`（本地环回）
