# 已训练模型应用测试流程详解

## 一、涉及的所有文件

### 1. 启动和配置文件

| 文件路径 | 作用 |
|---------|------|
| `drl-or-s/run.sh` | 启动脚本，包含测试命令和参数 |
| `drl-or-s/a2c_ppo_acktr/arguments.py` | 解析命令行参数（模式、拓扑、模型路径等） |

### 2. 核心程序文件

| 文件路径 | 作用 |
|---------|------|
| `drl-or-s/main.py` | DRL Agent 主程序，协调整个测试流程 |
| `drl-or-s/net_env/simenv.py` | 网络环境接口，与控制器和Mininet通信 |
| `drl-or-s/a2c_ppo_acktr/model.py` | 神经网络模型定义（Policy, GNNBase） |
| `drl-or-s/a2c_ppo_acktr/storage.py` | 经验回放缓冲区（RolloutStorage） |
| `drl-or-s/a2c_ppo_acktr/algo/ppo.py` | PPO算法实现（测试模式下不更新参数） |
| `drl-or-s/a2c_ppo_acktr/utils.py` | 工具函数（日志清理等） |

### 3. 控制器文件

| 文件路径 | 作用 |
|---------|------|
| `ryu-controller/controller.py` | Ryu SDN控制器，安装流表规则 |
| `ryu-controller/run.sh` | 启动控制器的脚本 |
| `ryu-controller/config_files/Abi.config` | 拓扑配置文件 |

### 4. 测试床文件

| 文件路径 | 作用 |
|---------|------|
| `testbed/testbed.py` | Mininet网络创建和流量生成 |
| `testbed/run.sh` | 启动测试床的脚本 |
| `testbed/client.py` | UDP客户端，在源主机发送流量 |
| `testbed/server.py` | UDP服务器，在目标主机接收流量并测量性能 |

### 5. 拓扑数据文件

| 文件路径 | 作用 |
|---------|------|
| `topology/Abi/Topology.txt` | 网络拓扑结构（节点、链路、带宽、丢包率） |
| `topology/Abi/TM.txt` | 流量矩阵（节点间流量需求分布） |
| `topology/Abi/link_weight.json` | 链路权重（用于基线算法对比） |

### 6. 模型文件

| 文件路径 | 作用 |
|---------|------|
| `drl-or-s/model/Abi-heavyload-gcn-sharepolicy-SPRsafe-mininet-2penalty-test-Abi-dynamic-extra2/agent0.pth` | 节点1的神经网络权重 |
| `drl-or-s/model/.../agent1.pth` | 节点2的神经网络权重 |
| ... | ... |
| `drl-or-s/model/.../agent10.pth` | 节点11的神经网络权重 |

### 7. 日志输出文件（运行时生成）

| 文件路径 | 作用 |
|---------|------|
| `drl-or-s/log/test/dist_type0.log` | 类型0流量的路径长度（vs最短路径） |
| `drl-or-s/log/test/demand_type0.log` | 类型0流量的需求满足率 |
| `drl-or-s/log/test/delay_type0.log` | 类型0流量的延迟 |
| `drl-or-s/log/test/throughput_type0.log` | 类型0流量的吞吐量 |
| `drl-or-s/log/test/loss_type0.log` | 类型0流量的丢包率 |
| `drl-or-s/log/test/globalrwd.log` | 全局奖励值 |
| `drl-or-s/log/test/circle.log` | 环路检测标志 |

### 阶段2：测试循环（每个请求的处理流程）

#### 2.1 生成路由请求

**文件**：`drl-or-s/net_env/simenv.py`

**流程**：
```
_update_state() 方法（第 400-500 行）
  ↓
移除过期请求（从 _request_heapq 中弹出）
  ↓
生成新请求
  ├─ 读取 TM.txt 流量矩阵
  ├─ 按概率采样源节点和目标节点
  ├─ 随机选择流量类型（rtype: 0-3）
  ├─ 随机选择需求带宽（demand）
  └─ 创建 Request 对象
  ↓
更新网络状态
  ├─ 计算链路可用带宽（link_capa - link_usage）
  ├─ 计算 widest path 信息
  └─ 构建观测向量（obs）
  ↓
返回 request 和 obses
```

**关键代码位置**：
- `drl-or-s/net_env/simenv.py` 第 400-500 行：_update_state

---

#### 2.2 DRL 模型计算路径

**文件**：`drl-or-s/main.py`, `drl-or-s/a2c_ppo_acktr/model.py`

**流程**：
```
main.py 第 248 行：envs.first_agent()
  ↓
从源节点开始，逐跳决策
  ↓
对每个智能体节点（main.py 第 250-271 行）：
  ├─ 获取当前观测 obs（网络状态）
  ├─ 构建图数据 Data(x, edge_index)
  ├─ 调用 actor_critic.act()（model.py 第 41-53 行）
  │   ├─ GNN 处理：提取节点特征
  │   ├─ Actor 网络：计算动作概率分布
  │   └─ 采样动作：选择下一跳邻居节点
  └─ 更新路径，继续到下一个智能体
  ↓
如果到达目标节点，路径完成
  ↓
返回完整路径：[0, 3, 5]（节点ID列表）
```

**关键代码位置**：
- `drl-or-s/main.py` 第 248-271 行：路径生成循环
- `drl-or-s/a2c_ppo_acktr/model.py` 第 41-53 行：act 方法
- `drl-or-s/net_env/simenv.py` 第 300-350 行：first_agent, next_agent

## 四、关键文件详细说明

### 1. drl-or-s/main.py（主程序）

**职责**：
- 解析命令行参数
- 初始化环境、模型、日志
- 主测试循环
- 协调所有组件

**关键方法**：
- `main()`：主函数，第 24-365 行
- 测试循环：第 229-320 行

**调用关系**：
```
main()
  ├─ get_args() → arguments.py
  ├─ NetEnv() → simenv.py
  ├─ Policy() → model.py
  ├─ PPO() → algo/ppo.py
  └─ RolloutStorage() → storage.py
```

---

### 2. drl-or-s/net_env/simenv.py（环境接口）

**职责**：
- 与 Mininet 和 Ryu 控制器通信
- 管理网络状态
- 生成路由请求
- 计算奖励

**关键方法**：
- `setup()`：初始化环境，加载拓扑（第 89-200 行）
- `reset()`：重置环境，生成首个请求（第 137-145 行）
- `step()`：执行一步，返回奖励和状态（第 183-350 行）
- `sim_interact()`：与控制器和Mininet交互（第 147-181 行）
- `_update_state()`：更新状态，生成新请求（第 400-500 行）
- `first_agent()`：获取首个智能体（第 300-320 行）
- `next_agent()`：获取下一个智能体（第 280-300 行）

**Socket 连接**：
- Mininet：127.0.0.1:5000（第 63-67 行）
- Ryu 控制器：127.0.0.1:3999（第 69-73 行）

---

### 3. drl-or-s/a2c_ppo_acktr/model.py（神经网络模型）

**职责**：
- 定义 Policy 网络结构
- 实现前向推理（act 方法）
- 支持 GNN 处理图数据

**关键类**：
- `Policy`：主策略网络（第 9-80 行）
- `GNNBase`：图神经网络基础（第 82-150 行）

**关键方法**：
- `act()`：推理动作（第 41-53 行）
  - 输入：图数据、条件状态、节点索引、类型索引
  - 输出：value（价值估计）、action（下一跳）、log_prob（动作概率）

---

### 4. ryu-controller/controller.py（SDN控制器）

**职责**：
- 管理 OpenFlow 交换机
- 接收 DRL 路径并安装流表
- 处理 ARP 和 PacketIn 事件

**关键方法**：
- `install_rules()`：DRL 路径接收服务（第 79-131 行）
- `add_flow()`：安装 OpenFlow 流表（第 156-175 行）
- `switch_features_handler()`：交换机连接处理（第 136-154 行）

### 单次请求的完整流程

```
时间 T0: DRL Agent 生成请求
  ├─ 文件：simenv.py::_update_state()

  ├─ 启动：client.py（源主机）

时间 T6: 测量性能
  ├─ 文件：server.py 第 36-55 行
```

## 十、故障排查

### 常见问题

1. **连接失败**
   - 检查端口是否被占用：`netstat -tulpn | grep 5000`
   - 检查启动顺序是否正确

2. **路径计算失败**
   - 检查模型文件是否存在
   - 检查拓扑文件是否正确

3. **流表未安装**
   - 检查控制器日志
   - 检查交换机 dpid 是否匹配

4. **性能测量异常**
   - 检查 client.py 和 server.py 是否正常启动
   - 检查流表是否正确安装

## 总结

已训练模型的应用测试涉及 **15+ 个文件**，主要分为：

1. **启动层**：run.sh 脚本
2. **DRL 层**：main.py, simenv.py, model.py
3. **控制器层**：controller.py
4. **测试床层**：testbed.py, client.py, server.py
5. **数据层**：Topology.txt, TM.txt, agent*.pth
6. **日志层**：各种 .log 文件

整个流程是一个**闭环反馈系统**：
```
请求生成 → DRL计算路径 → 控制器安装流表 → Mininet生成流量 → 
测量性能 → 计算奖励 → 记录日志 → 生成新请求 → ...
```

每个组件都有明确的职责，通过 Socket 和文件进行数据交换。
